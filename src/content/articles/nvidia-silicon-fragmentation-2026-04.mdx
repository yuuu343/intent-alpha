## TL;DR

A 1,000-job sample of NVIDIA's public Workday board on 2026-04-28 contains
**171 silicon and hardware-design roles**, plus another **86 in datacenter
networking** and **94 across automotive autonomy and robotics**. The composition
matters more than the total: NVIDIA is hiring chip-design organisations
*specialised by vertical*, not generalised GPU teams. The hiring shape is
inconsistent with a "single next-gen GPU" narrative and consistent with
multiple silicon SKUs targeted at distinct end-markets developed in parallel.

## What the cluster shows

Of 1,000 sampled postings:

| Cluster | Count | Examples |
|---|---|---|
| Silicon / HW design | **171** | ASIC Verification Engineer, Senior Silicon and System Product Lead, Senior Post Silicon Feature Development, Formal Verification Engineer, IC Post-Silicon Test Engineer |
| Datacenter & networking | **86** | Distinguished Software Engineer — NVLink Fusion Software, Software Engineer — NVLink GPU Networking, Principal Software Architect — Networking AI |
| Automotive / AV | **73** | (autonomous-driving stack roles, including Manager, Deep Learning — AV and Robotics) |
| Robotics / Omniverse / simulation | **21** | Machine Learning Engineer — Humanoid Robotics, Senior Software Engineer — Robotics, Senior TPM — Omniverse |
| Compilers / CUDA | **33** | (GPU kernel and compiler-side roles) |
| Foundation-model / training | **27** | (in-house model R&D) |

The **silicon cluster alone is 17.1% of the sample**. For a company whose
public identity is "GPUs and CUDA," the chip-design organisation looks unusually
heavy and unusually segmented — formal verification, post-silicon validation,
ASIC verification tooling, and *system-level* silicon product leadership all
appearing in parallel.

## Why this points to fragmentation, not consolidation

A monolithic next-gen GPU programme typically clusters hiring around a single
silicon timeline and a single architecture team. What's visible here is more
diffuse:

- **NVLink Fusion** is its own software stack (id `JR2000295`) — chip-to-chip
  fabric work distinct from rack-scale networking.
- **Networking AI** appears as its own architecture lane (id `JR2016684`) —
  embedded silicon in the networking layer, not just the compute layer.
- **Humanoid Robotics** has dedicated machine-learning engineering roles
  (id `JR2016809`) — the Isaac/robotics stack is no longer just simulation, it
  needs domain-trained models.
- The autonomous-driving and robotics teams are not converging into one
  "physical AI" group; they remain separate hiring lanes.

The shape is consistent with NVIDIA pursuing **silicon families per end-market**
— inference-class GPUs, AV-class SoCs, robotics-class compute, custom networking
ASICs — rather than a single "Blackwell+1" platform shipped to everyone.

## What would change our mind

Hypothesis flips to **low** confidence if any of the following hold over the
next two quarters:

- Silicon-design hiring concentrates back into a single product-line tag (e.g.,
  the Verification Engineer roles all reroute under one architecture umbrella).
- A unified physical-AI / robotics / AV organisational restructure consolidates
  the currently-separate hiring lanes.
- Public roadmap explicitly frames the next generation as a single platform
  for all verticals.

Hypothesis strengthens to **high** confidence if any of the following appear:

- A purpose-built networking-ASIC product is announced (the 86 networking
  hires materialise as silicon, not just software).
- A robotics-specific NVIDIA SoC ships with its own SDK, distinct from the
  general datacenter line.
- An automotive-class chip is announced separately from the datacenter
  generation cadence.

## Caveats

- Workday's public CXS API doesn't return department metadata, so this is a
  title-keyword cluster, not a department-cluster. Title regex misses cases
  where roles are tagged generally (e.g., "Senior Software Engineer" with
  silicon work hidden in the JD).
- 1,000 jobs is the per-fetch cap; the total NVIDIA open headcount is closer
  to 2,000 globally. The cluster ratios should be roughly preserved at scale,
  but absolute counts are conservative.
- "Silicon" hiring includes verification and validation roles that exist
  across any chip company; the diagnostic value is the *combination* with
  vertical-specific networking, robotics, and AV teams.
- We will revisit when the next snapshot lands.
